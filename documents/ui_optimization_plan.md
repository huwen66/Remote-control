# 客户端 UI 优化计划

## 现状分析

当前界面位于 [app.py:_build_home](file:///Users/zhou/Desktop/AI_tool/remote_control/app.py#L493-L628)，采用左右分栏布局，深色主题。

**存在的问题：**
1. 设备码（IP）和连接密码没有复制按钮，用户需手动选中复制
2. 信息层级不够清晰，关键数字不够突出
3. 缺少交互反馈（复制成功提示等）
4. 按钮和输入框样式偏基础，缺乏专业感
5. 权限状态和服务状态混在一起，不易区分

## 设计方向

**Visual Thesis**：冷静专业的远程控制工具，深色主题，克制的蓝色强调色，信息层级分明，操作直接高效。

**Interaction Thesis**：
- 复制按钮点击后显示「已复制」toast 提示
- 输入框聚焦时蓝色边框高亮过渡
- 按钮悬停时背景色微变 + 手形光标

## 改动清单

### 1. 左侧「本机信息」区域重构

**目标**：突出设备码和连接密码，增加复制功能

| 改动项 | 说明 |
|--------|------|
| IP 设备码行 | 右侧增加「复制」图标按钮，点击复制到剪贴板 |
| 连接密码行 | 右侧增加「复制」按钮和「刷新」按钮，复制到剪贴板 / 重新生成随机码 |
| 权限状态 | 单独一行，带圆点指示灯（绿/黄/灰），点击可重新检测 |
| 服务状态 | 单独一行，带圆点指示灯，与权限状态视觉区分 |

### 2. 右侧「连接到远程」区域优化

| 改动项 | 说明 |
|--------|------|
| 输入框样式 | 增加内边距，聚焦时蓝色边框高亮 |
| 连接按钮 | 更高、更醒目，圆角，悬停变色 |
| 错误提示 | 红色小字，距离输入框更近 |
| 底部提示文字 | 弱化，减少干扰 |

### 3. 全局优化

| 改动项 | 说明 |
|--------|------|
| Toast 提示 | 新增轻量 toast 组件，用于「已复制」「密码已更新」等反馈 |
| 间距系统 | 统一为 8/12/16/20/24 的 4px 基准 |
| 字体层级 | 标题 15px bold / 标签 10px / 数值 20px bold / 辅助文字 9px |

## 具体实现

### 3.1 复制功能

在 `App` 类中新增方法：
- `_copy_to_clipboard(text)` — 复制文本到剪贴板并显示 toast
- `_show_toast(message, duration=1500)` — 显示浮层提示

IP 和密码旁边各加一个「复制」按钮，调用对应复制方法。

### 3.2 密码刷新按钮

新增 `_refresh_code()` 方法，生成新的 6 位随机码，更新 `self._code` 和服务端 code，显示 toast。

### 3.3 布局调整

- 窗口宽度从 640 调整为 680，左侧 320，右侧 360
- 输入框 ipady 从 8 增加到 10
- 连接按钮 pady 从 10 增加到 12
- 密码字号从 18 增加到 22
- IP 字号从 14 增加到 16

### 3.4 Toast 组件

用 `Toplevel` 实现无边框浮层，位于窗口底部居中，1.5 秒后自动消失。深色半透明背景，白色文字。
